​ Briefing ​

Basis

Om de 60 seconden de fallback.json ophalen. 99% van de tijd is die leeg.
Hij kan echter 4 verschillende waarden bevatten

{}

Refresh

  • Op deze waarde vullen we een key in. Elke key is geldig voor 1 refresh.
  • Voor we de pagina herladen zullen we dus moeten kijken of de “key” al in localstorage zit.
  • Zo niet, slaan we de key op in localstorage en refreshen we de pagina. Waarna het hele systeem opnieuw begint.
  • De reden dat we die key opslaan in localstorage is zodat we niet in een refresh loop terecht komen
{
  "refresh": 1568972585,
}

Redirect

  • Op deze waarde vullen we een URL in
  • Voor we de redirect uitvoeren moeten we kijken of we nog niet op deze pagina zitten.
  • Als de redirect ≠ huidige pagina, sturen we de bezoeker door
{
  "redirect": "http://google.com",
}

Popup / Notification

  • Op deze waarde vullen we een HTML string in.
  • Als deze waarde ingevuld is dan plaats je een DIV op de pagina met de inhoud van die waarde.
  • Als de waarde verdwijnd in de JSON, verdwijnd de DIV ook.
{
  "popup": "<h1>Test</h1>",
}
